# GPT-2 Test

This document explains how to validate GPT-2 training on small subsets of prepared datasets.
The main idea is to intentionally overfit on a small training subset to confirm that the model can learn from the dataset.

## Usage

### 1. Prepare a dataset for training

Run the per-modality preparation script to build the batched training dataset:

```bash
python molcrawl/<task>/dataset/prepare_gpt2.py assets/configs/<task>.yaml
```

Examples:

```bash
python molcrawl/data/genome_sequence/dataset/prepare_gpt2.py assets/configs/genome_sequence.yaml
python molcrawl/data/protein_sequence/dataset/prepare_gpt2.py assets/configs/protein_sequence.yaml
python molcrawl/data/compounds/dataset/prepare_gpt2.py assets/configs/compounds.yaml
python molcrawl/data/molecule_nat_lang/dataset/prepare_gpt2.py assets/configs/molecule_nat_lang_config.yaml
python molcrawl/data/rna/dataset/prepare_gpt2.py assets/configs/rna.yaml
```

This step:

- loads the tokenized dataset,
- creates fixed-length batches (filling the full context window), and
- writes the training-ready dataset to disk.

### 2. Train GPT-2

Run:

`python molcrawl/models/gpt2/train.py molcrawl/models/gpt2/configs/<dataset>/train_gpt2_small_config.py`

Each dataset has a corresponding config that defines training parameters.

Example:

`python molcrawl/models/gpt2/train.py molcrawl/models/gpt2/configs/molecule_nat_lang/train_gpt2_large_config.py`

This starts training and writes checkpoints (for example, `out/ckpt.pt`).

### 3. Multi-GPU / DDP (optional)

If `torchrun` is available, you can run on multiple GPUs.

Single node, 4 GPUs:

`torchrun --standalone --nproc_per_node=4 config_file.py`

## Configuration Parameters

Set these in:

`path/to/corresponding/dataset/train_gpt2_config.py`

### Important Parameters

#### `tokenizer_path`

Some datasets require this parameter. In that case, set it to the tokenizer generated during data preprocessing.

- Genome Sequence:
  use `spm_tokenizer.model` generated by `molcrawl/data/genome_sequence/preparation.py`
  (path from `assets/configs/genome_sequence.yaml` -> `output_dir`)
- Compounds:
  use `vocab.txt` generated by `molcrawl/data/compounds/preparation.py`
  (path from `assets/configs/compounds.yaml` -> `vocab_path`)
- Molecule Natural Language:
  no setup required (`MoleculeNatLangTokenizer` is initialized internally in the config)
- Protein Sequence:
  no setup required (`EsmSequenceTokenizer` is initialized internally in the config)

#### `dataset_dir`

Path to the processed dataset (output of step 1 in [Usage](#usage)).
Set this to the dataset path printed by the preparation script.

### Logging

- `tensorboard`:
  enable TensorBoard logging for metrics (for example, loss and learning rate)
- `tensorboard_dir`:
  directory for TensorBoard logs

### Output

- `out_dir`:
  directory for checkpoints and logs

### Batch Settings

- `batch_size`:
  number of samples per batch per GPU
- `block_size`:
  maximum token sequence length
- `gradient_accumulation_steps`:
  number of accumulation steps before optimizer update

Effective batch size:

`batch_size × block_size × gradient_accumulation_steps × num_GPUs`

Example:

`8 × 1024 × 80 × 8 = 524,288 tokens`

### Learning Rate Schedule

- `max_iters`:
  total training iterations
- `lr_decay_iters`:
  iterations over which learning rate decays from `learning_rate` to `min_lr`
- `warmup_iters`:
  warmup iterations from 0 to `learning_rate`
- `learning_rate`:
  peak learning rate after warmup
- `min_lr`:
  minimum learning rate at the end of decay

### Evaluation and Logging

- `eval_interval`:
  evaluation frequency (iterations)
- `eval_iters`:
  number of evaluation batches
- `log_interval`:
  logging frequency (iterations)

### Regularization

- `weight_decay`:
  L2 regularization strength
