# GPT-2 モデル推論機能

## 概要

この機能は、GPT-2 トレーニングステータス Webインターフェースにインタラクティブな推論機能を追加します。ユーザーは任意のトレーニング済みモデルカードをクリックすることで、モデルの詳細を表示し、リアルタイムでテキストを生成するモーダルダイアログを開くことができます。

## コンポーネント

### 1. バックエンド API (`api/gpt2-inference.js`)

**エンドポイント:**

- `POST /api/gpt2-inference` - GPT-2 モデルで推論を実行
  - **リクエストボディ:**

    ```json
    {
      "dataset": "compounds|genome_sequence|protein_sequence|rna|molecule_nat_lang",
      "size": "small|medium|large|xl",
      "prompt": "入力テキストプロンプト",
      "maxLength": 128,
      "temperature": 1.0,
      "topK": 50,
      "numSamples": 1
    }
    ```

  - **レスポンス:**

    ```json
    {
      "success": true,
      "results": ["生成テキスト1", "生成テキスト2"]
    }
    ```

- `GET /api/gpt2-inference/config/:dataset` - デフォルト推論設定を取得
  - **レスポンス:**

    ```json
    {
      "success": true,
      "config": {
        "name": "Compounds",
        "start_token": "<|startofsmiles|>",
        "max_length": 128,
        "temperature": 1.0,
        "top_k": null,
        "example_prompts": ["<|startofsmiles|>C", "..."]
      }
    }
    ```

**機能:**

- HuggingFace 形式とレガシーチェックポイント形式の両方をサポート
- 各モデルの最新チェックポイントを自動検出
- データセット固有のデフォルト設定
- 既存の GPT モデルコードを使用した Python ベースの推論実行

### 2. フロントエンドモーダルコンポーネント (`src/InferenceModal.js`)

**Props:**

- `isOpen` (boolean) - モーダルの表示状態を制御
- `onClose` (function) - モーダルを閉じた際のコールバック
- `dataset` (string) - データセット識別子
- `size` (string) - モデルサイズ (small, medium, large, xl)
- `modelData` (object) - モデルのメタデータとチェックポイント情報

**機能:**

- **モデル情報の表示:**
  - トレーニングイテレーション数
  - バリデーションロス
  - モデルパラメータ（レイヤー数、埋め込み次元など）
  - 現在のステータス

- **インタラクティブな推論コントロール:**
  - サンプル候補付きテキストプロンプト入力
  - 調整可能な生成パラメータ:
    - 最大長 (32〜512 トークン)
    - 温度 (0.1〜2.0)
    - Top-K サンプリング (0〜200)
    - サンプル数 (1〜10)
- **結果表示:**
  - 複数の生成サンプル
  - クリップボードへのコピー機能
  - シンタックスハイライト付きフォーマット出力

### 3. トレーニングステータスコンポーネントの更新 (`src/GPT2TrainingStatus.js`)

**新機能:**

- ホバーエフェクト付きのクリック可能なモデルカード
- モーダルの状態管理
- 推論モーダルへのモデルデータの自動受け渡し

**ビジュアルインジケーター:**

- クリック可能なカードは紫色のボーダーで特別なホバーエフェクトを持ちます
- トレーニング済みモデルにカーソルを合わせるとポインターに変わります
- 未トレーニングのモデルはクリック不可のままです

### 4. スタイリング (`src/InferenceModal.css`)

**デザイン機能:**

- 紫テーマのモダンなグラデーションヘッダー
- レスポンシブレイアウト（モバイル対応）
- スムーズなアニメーションとトランジション
- カスタムスタイルのレンジ入力
- スクロール可能なコンテンツエリア
- 既存のインターフェースと調和したプロフェッショナルなカラースキーム

## 使用方法

### ユーザー向け

1. GPT-2 トレーニングステータスページに移動します
2. チェックポイントデータがあるモデルカードを探します（トレーニング統計が表示されます）
3. トレーニング済みのモデルカードをクリックします
4. 推論モーダルが開きます:
   - 上部にモデルの詳細
   - 中央に生成パラメータ
   - 推論を実行する「Generate」ボタン
5. 必要に応じてパラメータを調整します:
   - サンプルプロンプトを使ったクイックテスト
   - 創造性のための温度調整（低い値 = より決定論的）
   - サンプリングの多様性を制御する Top-K の設定
   - 生成するサンプル数の選択
6. 「Generate」をクリックして推論を実行します
7. 結果を確認し、必要に応じてクリップボードにコピーします

### データセット固有の設定

| データセット       | 開始トークン | 最大長 | 温度 | Top-K | ユースケース |
| ------------------- | ----------- | ------ | ---- | ----- | ------------ |
| Compounds (化合物)  | `<\|startofsmiles\|>` | 128 | 1.0 | なし | SMILES 生成 |
| Genome Sequence     | `<\|startoftext\|>` | 256 | 0.8 | 200  | DNA 配列 |
| Protein Sequence    | `<\|startoftext\|>` | 256 | 0.8 | 50   | アミノ酸配列 |
| RNA                 | `<\|startoftext\|>` | 256 | 0.8 | 100  | RNA 配列 |
| Molecule NL         | `<\|startoftext\|>` | 128 | 0.7 | 50   | 自然言語記述 |

## 技術的詳細

### チェックポイントの検出

システムは2つの形式のチェックポイントを自動検出します:

1. **HuggingFace 形式**（推奨）:
   - `checkpoint-XXXX` ディレクトリに格納
   - HuggingFace Transformers ライブラリを使用
   - 形式: `model.generate()` API

2. **レガシー形式**:
   - `ckpt.pt` ファイルとして格納
   - カスタム GPT モデルクラスを使用
   - 形式: カスタム `model.generate()` 実装

### Python インテグレーション

バックエンドは Python プロセスを生成して推論を実行します:

- パラメータを伴うPythonスクリプトを動的に生成
- 適切なモデルとトークナイザーを読み込む
- 両方のチェックポイント形式をシームレスに処理
- 結果を JSON 形式で返す

### エラーハンドリング

- モデルが見つからないエラー
- チェックポイントの読み込み失敗
- 推論実行エラー
- Python 環境の問題
- すべてのエラーはモーダルにわかりやすいメッセージで表示されます

## インストールとセットアップ

1. バックエンドの依存関係をインストールします:

   ```bash
   cd molcrawl-web
   npm install
   ```

2. Python の依存関係が利用可能であることを確認します:

   ```bash
   pip install torch transformers
   ```

3. `LEARNING_SOURCE_DIR` 環境変数を設定します:

   ```bash
   export LEARNING_SOURCE_DIR="learning_source"
   ```

4. サーバーを起動します:

   ```bash
   npm start
   ```

## トラブルシューティング

### モーダルが開かない

- ブラウザのコンソールでエラーを確認してください
- モデルに有効なチェックポイントがあることを確認してください
- データセット設定が存在することを確認してください

### 推論が失敗する

- Python 環境が正しく設定されていることを確認してください
- チェックポイントファイルが存在し、読み取り可能であることを確認してください
- 詳細なエラーメッセージについてはサーバーログを確認してください

### 生成が遅い

- GPU の有無が速度に大きく影響します
- 大きなモデルほど生成に時間がかかります
- `maxLength` や `numSamples` を減らすことを検討してください

## アーキテクチャ図

```
ユーザーがモデルカードをクリック
       ↓
GPT2TrainingStatus.js
       ↓ (モーダルを開く)
InferenceModal.js
       ↓ (POST リクエスト)
/api/gpt2-inference
       ↓ (プロセスを生成)
Python スクリプト
       ↓ (モデルを読み込む)
gpt2/model.py
       ↓ (生成)
結果 → InferenceModal → ユーザー
```

## セキュリティの考慮事項

- プロンプトの入力サニタイズ
- ハングを防ぐプロセスタイムアウト
- 生成のリソース制限
- 本番環境向けのアクセス制御
- API エンドポイントのレート制限

## パフォーマンスに関する注意事項

- モデルの読み込みにより、最初の生成は遅くなる場合があります
- 後続の生成は読み込み済みモデルを再利用します（HF 形式使用時）
- GPU アクセラレーションにより速度が大幅に向上します
- メモリ使用量はモデルサイズに比例します
